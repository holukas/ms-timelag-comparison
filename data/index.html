Data

ETH Zürich

All datasets live under data/ at the repository root, organized by processing stage. The raw 00-* inputs are tracked in git for provenance; the derived *.parquet files are gitignored (large and regenerable from the inputs). This page documents what the directory contains and how it maps onto the processing versions being compared.

Folders are numbered by processing stage. 00-* holds the raw inputs and is never written to; every higher-numbered folder is derived from it and can be regenerated by re-running the notebooks.

data/
  00-eddypro_fluxes_level-1/                  raw EddyPro flux output (input)
  00-eddypro_settings/                        EddyPro project + metadata (input)
  00-meteo/                                   meteorological data (input)
  01-eddypro_fluxes_level-1_parquet/          flux CSVs as Parquet         (notebook 01)
  02-eddypro_fluxes_level-1_parquet_subsets/  column subsets               (notebook 02)
  04-flux-product-2025.3_subset_2024/         FP2025.3 reference, 2024     (notebook 04)
  05-merged_variants_fluxproduct/             variants + product, merged   (notebook 05)

(There is no 03- data folder; notebook 03 produces figures, not a dataset.)

00-eddypro_fluxes_level-1/¶

EddyPro output in FLUXNET format (one row per half hour, ~9,500 records, 483 columns), one file per processing version. The version code is embedded in each filename (e.g. eddypro_QCL-1_…_fluxnet_…_adv.csv).

Columns central to this study:

  • Fluxes: FN2O, FCH4 (and quality/uncertainty companions such as FN2O_RANDUNC_HF, FCH4_RANDUNC_HF).

  • Time-lag diagnostics, per gas — the variable the comparison turns on: N2O_TLAG_USED, N2O_TLAG_ACTUAL, N2O_TLAG_NOMINAL, N2O_TLAG_MIN, N2O_TLAG_MAX and the equivalent CH4_TLAG_* set.

00-eddypro_settings/¶

The EddyPro configuration that produced each flux file: a .eddypro project file and its companion .metadata file per version. Filenames encode the time-lag strategy:

  • *_OPENLAG-10s_* — open covariance-maximization search (−1 to 10 s), no default fallback → versions *-1.

  • *_DEFAULT-10s_* — covariance maximization (10 s window) with a default fallback lag → versions *-2R.

  • *_CH-CHA_… — the earlier (2024) constant-lag runs → versions *-3.

These files are the auditable definition of each version; keep them in sync with the processing versions table.

00-meteo/¶

Meteorological data used as supporting context. (To be documented once populated.)

Derived stages (01-…, 02-…, 04-…, 05-…)¶

Generated by the notebooks; safe to delete and regenerate. All are gitignored Parquet.

  • 01-eddypro_fluxes_level-1_parquet/ — each flux CSV read and re-saved as Parquet, one file per version (LGR-1.parquet, …). Produced by notebook 01.

  • 02-eddypro_fluxes_level-1_parquet_subsets/ — column subsets of the above (fluxes + time-lag diagnostics). Produced by notebook 02.

  • 04-flux-product-2025.3_subset_2024/ — the year-2024 subset of the final flux product FP2025.3 (CH-CHA, 2005–2024), loaded from an external location and used as the independent reference. Produced by notebook 04.

  • 05-merged_variants_fluxproduct/ — all six variant subsets merged with the FP2025.3 2024 reference on the timestamp index, each source’s columns suffixed by origin (FN2O_LGR-1, FN2O_FP2025.3, …). Produced by notebook 05.
